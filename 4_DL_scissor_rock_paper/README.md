## 가위바위보 이미지를 통한 분류기 만들기

### 1. 실험 목표
- 직접 촬영한 가위바위보 사진을 학습시켜, 가위, 바위, 보를 정확하게 분류할 수 있는 모델을 구축한다.
- 훈련 데이터와 테스트 데이터 사이의 차이를 극복하고 일반화된 성능을 확보하는 과정을 실험한다.

### 2. 실험 정보
- 훈련 데이터 <br>
1. 수량: 총 300장 (가위: 100, 바위: 100, 보: 100)
2. 특징: 웹캠을 통해 촬영된 오른손 이미지
3. 전처리: 28x28 사이즈로 Resize 및 0~1 사이 값으로 정규화<br>

- 테스트 데이터
1. 수량: 총 487장 (가위: 178, 바위: 162, 보: 147)
2. 특징: 웹캠을 통해 촬영된 이미지
3. 전처리: 훈련 데이터와 동일하게 28x28 Resize 및 정규화 진행<br>

- 사용한 모델 (CNN)
1. Conv2D: 필터 32개, 커널 (3,3), ReLU 활성화
2. MaxPooling2D: (2,2)
3. Conv2D: 필터 32개, 커널 (3,3), ReLU 활성화
4. MaxPooling2D: (2,2)
5. Flatten: 데이터 1차원화
6. Dense: 32개 노드, ReLU 활성화
7. Dense (Output): 3개 노드, Softmax 활성화

### 실험 결과
- Test Loss: 4.4707
- Test Accuracy: 0.3368 (약 33.7%)

### 결과 분석
- 과적합(Overfitting) 발생: 훈련 데이터에 대해서는 매우 높은 정확도(98%)를 보였으나, 테스트 데이터에서는 약 33.7%로 무작위 확률 수준의 성능을 기록함.
- 불일치: 훈련에 쓰인 손(왼손)과 테스트에 쓰인 손(오른손)의 형태적 차이가 모델의 예측 능력에 큰 영향을 미침.

### 회고
- 이미지 해상도 저하: 28x28 사이즈는 손가락의 미세한 형태를 파악하기에 정보량이 너무 부족함.
- 데이터의 한계: 훈련 이미지의 종류가 비슷비슷하고 양이 적어(300장) 모델이 다양한 상황을 학습하지 못함.
- 손의 방향성 문제: 오른손으로만 학습한 모델이 테스트 이미지의 특징을 찾아내는 데 부족함이 있던 거 같음.
